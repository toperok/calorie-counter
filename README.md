# Предсказание калорийности блюд (Multi-modal Regression)

Проект по созданию нейросети, которая предсказывает калорийность блюда на основе двух типов данных: **фотографии** и **текстового описания** его состава. Это решение объединяет компьютерное зрение (CV) и обработку естественного языка (NLP).

## Описание задачи

Задача: реализовать и обучить мультимодальную модель, которая предсказывает числовое значение (калории). Модель может быть полезна в fitness-трекерах и health-приложениях для автоматизации контроля питания.

### Этапы работы:

1. **Исследовательский анализ (EDA)**: Изучение распределения данных, визуализация и выбор стратегии аугментации.
2. **Реализация пайплайна**: Создание модульной структуры кода в `.py` файлах.
3. **Обучение**: Запуск тренировочного цикла (40 эпох) с использованием GPU и логированием результатов.
4. **Валидация**: Тестирование модели, расчет метрик (MAE, RMSE, MAPE) и анализ «тяжелых» примеров.

# Метрики качества

Для оценки качества предсказация калорийности  использовал:

* MAE (Mean Absolute Error) показывает среднюю ошибку в абсолютных единицах (калориях). Если MAE = 50, это значит, что в среднем модель ошибается на 50 ккал в обе стороны. Она проста и понятна.
* RMSE (Root Mean Squared Error) работает строже. Она возводит ошибки в квадрат перед усреднением, поэтому сильнее штрафует за крупные промахи. Если модель один раз ошиблась на 500 ккал, RMSE будет намного выше, чем MAE. Это полезно, если для вас критично не допускать огромных «ляпов» в диете.
* MAPE (Mean Absolute Percentage Error) измеряет ошибку в процентах. Ошибка в 100 ккал для соуса (300 ккал) — это много, а для целого дня питания (2500 ккал) — мелочь. MAPE помогает понять относительную точность: например, «модель ошибается в среднем на 10% от веса блюда».

* **Язык**: Python
* **Библиотеки**: PyTorch, Transformers, Timm, Pandas, Scikit-learn

   ```
   bash

   # Для macOS / Linux

   ```

   Используйте `uv` для автоматической установки Python и всех необходимых зависимостей.
